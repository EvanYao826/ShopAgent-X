## 一、项目概述

ShopAgent-X 是一个**对话式电商导购 AI Agent**，以聊天为主界面，AI 自动推荐商品并嵌入商品卡片。

**技术栈:**

| 层级 | 技术 | 端口 |
|------|------|------|
| App 端 | Android (Kotlin + Jetpack Compose) | - |
| Java 后端 | Spring Boot 3.2 + MyBatis-Plus | 8080 |
| Python AI 服务 | FastAPI + LangChain | 8000 |
| 向量数据库 | Milvus 2.4+ / FAISS (备选) | 19530 |
| 缓存 | Redis 7.0 | 6379 |
| 关系数据库 | MySQL 8.0 | 3306 |

**架构简图:**

```
App 端 ──HTTP──→ Java 后端 ──REST──→ Python AI 服务
                  │                      │
                  ├─ MySQL (业务数据)      ├─ Milvus (向量检索)
                  ├─ Redis (缓存/会话)     ├─ MySQL (商品查询)
                  └─ Caffeine (本地缓存)   └─ FAISS (备选向量库)
```

### 各页面详细说明

**Tab 1 — 对话页**
- 类豆包对话界面，消息气泡流式渲染
- AI 回复文字 + 嵌入商品卡片（图片+标题+价格+评分）
- 支持多轮追问："有没有更便宜的？""这款和XX比怎么样？"
- 支持拍照发图（调用图片上传接口）

**Tab 2 — 商品列表页**
- 顶部搜索栏（支持文字搜索 + 拍照搜图入口）
- 分类标签横向滚动（护肤品/彩妆/身体护理 等）
- 商品网格卡片（图片+标题+价格+评分+销量）
- 右下角悬浮购物车图标 🛒 + 角标数量
- 点击商品卡片 → 进入商品详情页

**商品详情页（Tab 2 子页面）**
- 商品图片轮播
- 基本信息（标题/品牌/价格/评分/销量）
- SKU 选择（规格/颜色，点击弹出选择器）
- FAQ 折叠列表
- 评论列表（分页）
- 底部操作栏：[加入购物车] 按钮
- 右下角悬浮购物车图标 🛒 + 角标

**购物车页**
- 购物车商品列表（商品图片+标题+SKU+价格+数量加减）
- 左滑删除
- 底部合计金额 + 结算按钮（结算按钮可不实现实际支付，点击弹提示即可）
- 空车状态："购物车是空的，去逛逛吧"

**Tab 3 — 我的页**
- 头像 + 用户名 + 手机号
- 收藏列表入口 → 收藏列表页（商品卡片列表）
- 浏览历史入口 → 浏览历史页（商品卡片列表+时间）
- 右上角设置齿轮 → 修改个人信息 / 退出登录

**登录/注册页**
- 手机号 + 密码登录
- 手机号 + 验证码注册

## 三、后端接口文档（完整）

### 统一返回格式 Result<T>

所有接口统一使用 `Result<T>` 包装返回，App 端按此格式解析：

```kotlin
// Kotlin 数据类定义
data class Result<T>(
    val code: Int,        // 200=成功, 其他=失败
    val message: String,  // "success" 或错误描述
    val data: T?          // 业务数据，失败时为 null
)

// 成功示例
{ "code": 200, "message": "success", "data": { ... } }

// 失败示例
{ "code": 401, "message": "未登录或Token已过期", "data": null }
{ "code": 400, "message": "参数错误: userId不能为空", "data": null }
{ "code": 500, "message": "服务器内部错误", "data": null }
```

**App 端统一处理建议:**
```kotlin
if (result.code == 200) {
    // 成功，使用 result.data
} else if (result.code == 401) {
    // Token 过期，调用 /api/auth/refresh 刷新，失败则跳转登录页
} else {
    // 其他错误，Toast 提示 result.message
}
```

### 3.2 对话模块 ChatController

**Base URL:** `http://{HOST}:8080/api/chat`

#### 创建对话
```
POST /conversations?userId={userId}&title={可选}
Response: Result<Conversation>
→ { "id": 1, "userId": 1, "title": "新对话", "createTime": "..." }
```

#### 获取对话列表
```
GET /conversations?userId={userId}
Response: Result<List<Conversation>>
```

#### 发送消息（核心接口，非流式）
```
POST /messages
Body: { "userId": 1, "conversationId": 1, "content": "推荐一款适合油皮的精华" }
Response: Result<Message>
说明: 非流式接口，等待 AI 完整回复后一次性返回。如需流式体验，请用下方 SSE 接口
```

#### 发送消息（SSE 流式，推荐）
```
POST /stream/messages
Header: Accept: text/event-stream
Header: Authorization: Bearer ***
Body: { "userId": 1, "conversationId": 1, "content": "推荐一款适合油皮的精华" }
Response: SSE 事件流

SSE 事件格式:
  data: {"type":"content","content":"为您"}
  data: {"type":"content","content":"推荐"}
  data: {"type":"product_cards","product_cards":[...]}
  data: {"type":"done"}

说明: Java 后端透传 Python 服务的 SSE 流，App 端用 OkHttp EventSource 接收
```

#### 获取历史消息
```
GET /messages?conversationId={conversationId}
Response: Result<List<Message>>
→ 每条 assistant 消息都包含 productCards（如果有）
```

#### 删除对话
```
DELETE /conversations/{id}
Response: Result<String>
```

#### 更新对话（置顶等）
```
PUT /conversations/{id}
Body: { "title": "新标题", "isPinned": true }
Response: Result<Conversation>
```

#### 上传图片（拍照搜图用）
```
POST /upload/image
Content-Type: multipart/form-data
Body: file={图片文件}
Response: Result<Map>
→ { "id": "uuid", "name": "photo.jpg", "url": "/api/chat/view/image/uuid" }
```

#### 查看图片
```
GET /view/image/{id}
Response: 图片二进制流
```

#### 消息反馈
```
POST /messages/feedback
Body: { "messageId": 1, "feedbackType": "like" }  // like / dislike
```

### 3.6 购物车模块 CartController（新增）

**建表 SQL:**
```sql
CREATE TABLE `cart` (
  `id`          BIGINT   NOT NULL AUTO_INCREMENT COMMENT '购物车项ID',
  `user_id`     BIGINT   NOT NULL                COMMENT '用户ID',
  `product_id`  BIGINT   NOT NULL                COMMENT '商品ID',
  `sku_id`      BIGINT   DEFAULT NULL            COMMENT 'SKU ID（可为空，无规格商品）',
  `quantity`    INT      DEFAULT 1               COMMENT '数量',
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '加入时间',
  `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_user_product_sku` (`user_id`, `product_id`, `sku_id`),
  KEY `idx_user_id` (`user_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='购物车表';
```

## 五、对话式导购核心流程

### 5.1 普通对话流程

```
用户发消息 "推荐一款适合油皮的精华"
    │
    ▼
App 端 POST /api/chat/messages { content: "推荐一款适合油皮的精华", userId, conversationId }
    │
    ▼
Java ChatController → ChatService.sendMessage()
    │
    ├─ 保存用户消息到 message 表
    │
    ▼
Java AiService → POST http://localhost:8000/api/ask
    │
    ▼
Python RouterAgent.classify_task()
    │
    ├─ IntentType.SHOPPING → ShoppingAgent.recommend()
    │     │
    │     ├─ 1. 读取会话记忆 (conversation_memory_read)
    │     ├─ 2. 向量检索相关商品知识 (vector_store.search, k=8)
    │     ├─ 3. MySQL 模糊匹配商品 (LIKE title/brand/tags)
    │     ├─ 4. LLM 生成推荐话术
    │     ├─ 5. 构建商品卡片 (最多5个)
    │     └─ 6. 保存对话记忆
    │
    ├─ IntentType.CHITCHAT → ChitChatAgent.chat() (无商品卡片)
    └─ IntentType.KNOWLEDGE_QA → KnowledgeQAAgent.ask() (无商品卡片)
    │
    ▼
返回 { answer, sources, taskType, productCards }
    │
    ▼
Java AiServiceImpl → 解析响应 → AiResponse { answer, productCards, sources, taskType }
    │
    ▼
ChatServiceImpl → 保存 AI 消息到 message 表 (含 productCards JSON)
    │
    ▼
返回 Message 给 App 端
    │
    ▼
App 端渲染: 文字气泡 + 商品卡片列表
```

### 5.2 流式对话流程（推荐，体验更好）

```
App 端 POST /api/chat/stream/messages (SSE)
    │
    ▼
Java 后端透传 → Python POST /api/ask/stream
    │
    ▼
Python 返回 SSE 事件流:
    data: {"type": "routed", "task_type": "shopping"}
    data: {"type": "token", "content": "为您"}
    data: {"type": "token", "content": "推荐"}
    data: {"type": "token", "content": "以下"}
    ...
    data: {"type": "product_cards", "product_cards": [...], "sources": [...]}
    data: {"type": "end", "content": "完整答案", "task_type": "shopping"}
```

**SSE 事件类型:**

| type | 说明 | App 端处理 |
|------|------|-----------|
| `routed` | 意图识别完成 | 可选：显示"正在为您查找..." |
| `token` | 文字流式输出 | 逐字追加到消息气泡 |
| `product_cards` | 商品卡片数据 | 渲染商品卡片列表 |
| `answer` | 完整答案 | 替换流式拼接的内容 |
| `end` | 流结束 | 关闭 SSE 连接 |
| `error` | 错误 | 显示错误提示 |

## 八、Agent 架构说明

### 8.1 意图分类（RouterAgent 路由逻辑）

```
用户输入
    │
    ▼
IntentClassifier.classify()
    │
    ├─ SHOPPING      → ShoppingAgent (商品推荐/比价/搜索)
    ├─ CHITCHAT      → ChitChatAgent (闲聊/问候)
    ├─ KNOWLEDGE_QA  → KnowledgeQAAgent (知识问答)
    │                    │
    │                    ├─ 简单问题 → L1 直接检索+生成
    │                    ├─ 中等问题 → L2 问题改写+检索+重排
    │                    └─ 复杂问题 → L3 升级为 ReasoningAgent
    ├─ ADMIN_COPILOT → AdminCopilotAgent (管理端助手)
    └─ KNOWLEDGE_INSPECTION → InspectionAgent (知识库巡检)
```

### 8.2 ShoppingAgent 执行流程

```
1. 读取会话记忆 → 获取历史对话上下文
2. 向量检索 → Milvus/FAISS 语义搜索 (k=8, threshold=0.6)
3. MySQL 模糊匹配 → LIKE title/brand/tags/sub_category
4. 构建商品上下文 → 拼接商品信息给 LLM
5. LLM 生成推荐话术 → 亲切自然的语气
6. 构建商品卡片 → 最多5个，含 product_id/title/brand/price/image_url/rating
7. 保存对话记忆 → 写入 Redis
```

### 8.3 问题复杂度分级

| 级别 | 条件 | 处理链路 |
|------|------|---------|
| L1 简单 | 普通问题，无代词 | 直接检索 + 生成 |
| L2 中等 | 含代词("它""这个") 或 短问题(<10字) | 问题改写 + 检索 + 重排序 + 生成 |
| L3 复杂 | 含对比/分析词 且 长问题(>15字) | 分解子问题 + 逐个推理 + 汇总 |

## 十一、注意事项

1. **所有 API 统一用 `Result<T>` 包装返回**，格式: `{ "code": 200, "message": "success", "data": {...} }`（详见第三章开头的 Result 定义）
2. **JWT Token** 放在 `Authorization: Bearer {accessToken}` 请求头，accessToken 过期用 refreshToken 调用 `/api/auth/refresh` 刷新
3. **商品卡片** 只在 `taskType = "shopping"` 时返回，其他类型 productCards 为空数组
4. **图片 URL** 如果是相对路径，App 端需要拼接 `http://{HOST}:8080` 前缀
5. **SSE 流式接口** 需要设置 `Accept: text/event-stream`，用 OkHttp 的 EventSource
6. **历史消息** 中的 productCards 是 JSON 字符串存在 DB，取出来已经是 List<Map>，直接用
7. **购物车** 同一商品+同一SKU 重复加入时数量累加，不是新增一条记录（UNIQUE KEY 约束）
8. **收藏/浏览历史** 的商品信息由后端关联查询返回，App 端不需要二次请求商品详情
